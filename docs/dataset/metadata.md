# Metadata

Qortex reads sidecars, event tables, participants, and root metadata before it downloads large imaging files. This is the cheapest way to learn whether a dataset has labels, usable demographics, task timing, TR, channel metadata, or companion files.

## Real Participants Example

This figure is generated by `scripts/generate_docs_examples.py` from public OpenNeuro dataset `ds000001`:

```python
from qortex import Dataset

ds = Dataset("ds000001")
participants = ds.participants(prefer_api=False)
print(participants.shape)
print(participants.head())
```

## Dataset-Level Metadata

```python
from qortex import Dataset

ds = Dataset("ds000001")

metadata_files = ds.metadata_files()
participants = ds.participants()
description = ds.preview("dataset_description.json")
```

`participants()` returns a Polars dataframe. Qortex tries OpenNeuro API summary metadata first; pass `prefer_api=False` to read `participants.tsv` directly from the CDN.

## File-Level Sidecars

```python
sidecar = ds.sidecar(
    "sub-01/func/sub-01_task-balloonanalogrisktask_run-01_bold.nii.gz"
)
events = ds.events(subject="01", task="balloonanalogrisktask", run="01")

print(sidecar["RepetitionTime"])
print(events.select(["onset", "duration", "trial_type"]).head())
```

`sidecar()` accepts a BIDS-relative path or file record. It merges JSON sidecars through BIDS inheritance where possible. `events()` fetches the matching `events.tsv`; pass `subject`, `session`, `task`, and `run` to disambiguate.

## CLI

List metadata and sidecar/table files:

```bash
qortex metadata ds000001 --limit 20
```

Download metadata only:

```bash
qortex metadata ds000001 --download --output-dir data/ds000001-meta
```

Preview a specific remote file:

```bash
qortex preview ds000001 participants.tsv --rows 5
qortex preview ds000001 sub-01/func/sub-01_task-balloonanalogrisktask_run-01_events.tsv --rows 5
```

## What Metadata Can Decide

| Metadata | What it answers |
|---|---|
| `dataset_description.json` | Dataset name, BIDS version, license, generated-by metadata. |
| `participants.tsv` | Subject IDs, age/sex/group columns when provided. |
| `*_events.tsv` | Candidate label columns, onsets, durations, class counts, trial timing. |
| JSON sidecars | TR, task name, sampling frequency, units, slice timing, acquisition metadata. |
| `channels.tsv` | Channel names, types, units, and bad-channel flags. |
| `bval` / `bvec` | DWI shell structure and gradient companions. |

This information is often enough to decide whether the full dataset is worth downloading.
